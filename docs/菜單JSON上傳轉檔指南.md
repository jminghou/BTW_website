# 數位看版 — 菜單 JSON 上傳自動轉檔指南

> 本文件說明：如何**直接上傳餐點資料 JSON**，由系統自動轉成菜單 HTML 並存成素材。
> 適用對象：日常維護／上稿人員。最後更新：2026-05-19。

---

## 一、這個功能在做什麼？

以前的流程是：用外部程式把資料轉成菜單 HTML → 再上傳。
現在可以**直接上傳 JSON，系統一步到位**幫你：

1. **自動拆分** — 一份含整月的 JSON，自動依「據點＋時段＋日期」拆成每日菜單
2. **自動轉檔** — 每日資料轉成你熟悉的菜單 HTML 版型
3. **自動存檔** — 每份菜單存成一個素材，命名如 `F3_L_2026-05-25.html`

> 💡 你之前「拆分單日 + 調整格式」的程式可以退休了 — 這些都內建在轉檔功能裡。

---

## 二、操作流程

1. 進入 **後台 → 電子看版 → 素材**
2. 上傳區點上方頁籤 **「上傳 JSON 自動轉檔」**
3. 選擇 **所屬廠區**
4. 選擇 JSON 檔（**可一次選多個**）
5. 按 **「⚙ 轉檔並上傳」**
6. 完成後會列出產生的菜單檔名（例：成功產生 31 個菜單）

接著就照常 **建立播放清單 → 設定排程 → 螢幕播放**。

---

## 三、可以上傳哪種 JSON？

**兩種都可以，系統都會自動處理：**

| JSON 類型 | 範例 | 結果 |
|---|---|---|
| **完整月匯出檔**（含整月、132 筆） | `世界先進三廠 fab3_2026年5月午餐菜單匯出_xxx.json` | 自動拆成 31 個單日菜單 |
| **單日檔**（單日、數筆） | `世界先進三廠 fab3_午餐_05-25.json` | 產生 1 個菜單 |

**直接上傳完整月匯出檔最方便** — 不必再自己拆分。

---

## 四、JSON 需要哪些欄位？

每筆餐點資料是一個物件，必須包含以下欄位（與系統匯出的格式相同，通常不用調整）：

| 欄位 | 用途 | 範例 |
|---|---|---|
| `餐點名稱` | 菜名（中） | 牛肉咖哩蛋包飯 |
| `英文名稱` | 菜名（英） | Beef Curry Omelet Rice |
| `售價` | 價格 | 125 |
| `圖片網址` | 餐點照片完整網址 | https://www.bythewaylife.com.tw/... |
| `餐廳名稱` | 餐廳（用於分組標題） | 義式就好 |
| `據點` | 廠區 | 世界先進三廠 fab3 |
| `時段` | 早餐／午餐／晚餐／宵夜 | 午餐 |
| `日期` | YYYY-MM-DD | 2026-05-25 |
| `星期` | 星期一 ～ 星期日 | 星期一 |

格式範例：
```json
[
  {
    "餐點名稱": "牛肉咖哩蛋包飯",
    "英文名稱": "Beef Curry Omelet Rice",
    "售價": "125",
    "圖片網址": "https://www.bythewaylife.com.tw/btwFile/Meals/xxx.png",
    "餐廳名稱": "義式就好",
    "據點": "世界先進三廠 fab3",
    "時段": "午餐",
    "日期": "2026-05-25",
    "星期": "星期一"
  }
]
```

## 五、自動轉換規則（不用記，系統會做）

| 來源 | 轉成 |
|---|---|
| `據點`「世界先進三廠 fab3」 | 側欄「FAB 3」、檔名「F3」 |
| `時段`「午餐」 | 側欄「午餐」、檔名「L」（早B／午L／晚D／宵N） |
| `星期`「星期一」 | 側欄「MON」 |
| `日期`「2026-05-25」 | 側欄「2026.05.25」、檔名日期 |
| 相同 `餐廳名稱` 的餐點 | 自動歸在同一個餐廳標題下 |
| 餐點順序 | 自動編號（大圖區編號圖 01～10） |

**拆檔規則**：同一個「據點＋時段＋日期」的所有餐點 → 合成一份菜單。
不同日期、不同時段 → 各自一份菜單。

---

## 六、限制與注意事項

| 項目 | 說明 |
|---|---|
| **單日餐點上限** | 大圖區編號圖只有 01～10。**單日超過 10 道**時，第 11 道起的編號圖會缺（系統會在結果列出 ⚠ 警告） |
| **密集模式** | 單日超過 6 道時，右側列表自動縮小間距以容納 |
| **缺圖** | 若某筆 `圖片網址` 空白，該餐點顯示「暫缺圖片」預設圖 |
| **必填欄位** | 缺 `餐點名稱`／`據點`／`時段`／`日期` 任一，整份 JSON 會報錯不處理 |
| **檔名格式** | 只有 `世界先進一/二/三廠 fabX` 會轉成 F1/F2/F3；其他據點檔名前綴會空白 |

## 七、常見問題

| 症狀 | 原因 | 解法 |
|---|---|---|
| 「JSON 格式錯誤，無法解析」 | 檔案不是合法 JSON | 用編輯器確認 JSON 格式正確（成對括號、逗號） |
| 「每筆資料都必須有…欄位」 | 缺必填欄位 | 補齊 餐點名稱／據點／時段／日期 |
| 產生的菜單數比預期少 | 多筆資料的日期/時段相同被併在一起 | 正常行為，同日同時段就是一份 |
| 某餐點沒有編號圖 | 該日超過 10 道 | 拆成兩個時段，或接受無編號（不影響其他資訊） |
| 預覽是裸 HTML 無樣式 | 共用資源未部署 | 見[新增看版版型指南](新增看版版型指南.md)的資源部署說明 |

---

## 八、跟「直接上傳 HTML」的差別

素材頁有兩種上傳模式，依需求選擇：

| 模式 | 適用 |
|---|---|
| **上傳 JSON 自動轉檔** | 有餐點資料 JSON，想讓系統產生菜單（**日常推薦**） |
| **直接上傳 HTML** | 已經有現成的菜單 .html 檔（或非標準版型的特製頁面） |

兩種產生的素材完全一樣，後續播放清單／排程操作沒有區別。
